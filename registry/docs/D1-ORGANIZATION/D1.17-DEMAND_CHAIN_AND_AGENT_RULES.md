# UNIVERSAL AGENT RULES — Applies to ALL Agents

## Session Protocol
1. **Session START** → Read MC board for your assigned tasks first. Pick highest priority queued/in-progress task. Read last comment block — continue from there.
2. **Session END** → PUT timestamped comment on your task: done/remaining/blockers/next pickup point.
3. **One task = one session.** Too big? Dept leader breaks it smaller.
4. **No two agents on the same task simultaneously.**

## Demand Chain
- Captain demands from → Nami (strategic)
- Nami demands from → Jinbe (process) + Usopp (quality)
- Usopp demands from → Dept Leaders (via stories with DoD)
- Dept Leaders demand from → Sub-agents (via tasks)
- Each layer demands accountability from below, presents results up.

## Document Map — What Goes Where
- **SOUL.md** → Personality ONLY. ~50 lines. Nothing else.
- **AGENTS.md** → Permanent rules, scope, department structure. The constitution.
- **TOOLS.md** → Environment data. IPs, paths, API endpoints.
- **MC Task notes** → Current work context. Session comments. Pickup instructions.
- **HEARTBEAT.md** → Heartbeat checks only. Not loaded in task sessions.
- **memory/YYYY-MM-DD.md** → Daily log. Written at session end.
- **MEMORY.md** → Long-term curated memory. Main session only.

## Skill Self-Install
- Agents CAN install OpenClaw skills within their department scope.
- Check available skills: look at skills/ directory or clawhub.com
- Only install skills relevant to your work. Don't install skills outside your scope.
- If unsure whether a skill fits your scope: ask your department leader.

## File Access Rules
- ONLY edit files in YOUR workspace.
- Shared files (shared-memory/) → append-only with your agent name + timestamp.
- NEVER rewrite another agent's files.
- If you need work done in another department → create an MC task assigned to them.

## Captain-Direct Orders
- If Captain gives you work directly (bypassing the pipeline): 
  1. Acknowledge immediately
  2. Create an MC task on the board
  3. Execute and document as you go
  4. Notify Nami when done

## Task Board
- MC API: http://127.0.0.1:18800/api/tasks
- Every task MUST belong to a story (unless super-small sub-agent work)
- Completed stories with full task logs = reusable skills
- Nothing exists outside the board

## Agent-to-Agent Communication — OFFICIAL METHOD (2026-03-24)
**DO NOT use sessions_send for inter-agent comms.** It hits stale main sessions.

**USE one-shot crons with isolated sessions:**
```
cron(action=add, job={
  agentId: "target-agent",
  sessionTarget: "isolated",
  deleteAfterRun: true,
  schedule: { kind: "at", at: "<1 min from now>" },
  payload: { kind: "agentTurn", message: "your message here", timeoutSeconds: 120 },
  delivery: { mode: "announce", channel: "telegram", to: "-1003814818004:topic:1452" }
})
```

Why: Creates fresh isolated session → agent reads message with clean context → responds properly.
Delivers to Telegram so Captain can see the exchange.
