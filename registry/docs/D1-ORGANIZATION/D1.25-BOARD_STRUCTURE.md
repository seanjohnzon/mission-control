# D1.25 — Board Structure

MC Task Board = our database. Four views:

1. **Backlog** (was "Tasks") — everything that needs doing, Captain-direct items
2. **Sprint Board** — what's actively being worked on NOW
3. **Projects Board** — big goals, epic hierarchy
4. **Improvements** — research findings, suggestions, ideas (Robin/Brook dump here, Usopp pulls when creating stories)

## Rules
- Every task MUST belong to a story (unless super-small sub-agent work)
- Completed stories with full task logs = reusable skills
- Nothing exists outside the board
- MC API: http://127.0.0.1:18800/api/tasks
