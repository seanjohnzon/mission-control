# SESSION PROTOCOL — Universal (Goes in EVERY department's AGENTS.md)

## DRAFT — For Captain Review Before Applying

### Session Start — What To Do First
1. **Direct message?** → Execute that message immediately. It IS your task.
2. **Cron/heartbeat session?** → Check MC board for your assigned tasks:
   ```
   curl -s http://127.0.0.1:18800/api/tasks | python3 -c "
   import sys,json
   tasks=json.load(sys.stdin)['tasks']
   mine=[t for t in tasks if t.get('assigned','').lower()=='YOUR_NAME' and t.get('status') in ['queued','in-progress']]
   for t in sorted(mine, key=lambda x: x.get('priority','P9')):
       print(f\"{t['id']}: [{t['status']}] {t['title']}\")
   "
   ```
3. **Pick highest priority task** that is queued or in-progress
4. **Read the task's FULL notes** — the last block is where the previous session left off
5. **Continue from there** — don't restart, don't duplicate work

### Session End — What To Do Before Stopping
Before your session ends (timeout, compaction, or task complete), you MUST:

```
curl -s -X PUT http://127.0.0.1:18800/api/tasks/TASK_ID -H "Content-Type: application/json" -d '{
  "notes": "EXISTING NOTES...\n\n---\n[YYYY-MM-DD HH:MM] agent:NAME — SESSION END\n✅ Done: what was completed\n⬜ Remaining: what is left\n🚫 Blockers: none or describe\n📍 Next: exact pickup point for next session"
}'
```

If task is COMPLETE: mark status "completed" with evidence in notes.
If task is BLOCKED: mark status "blocked", describe blocker, escalate.
If task is IN-PROGRESS: leave it in-progress with clear pickup instructions.

### One Task = One Session
- Each task should be completable in ONE session
- If a task is too big: that's a problem for the dept leader to solve
  - Break it into smaller tasks
  - Or retrigger a new session with context from the last comment
- NEVER have two agents working on the same task simultaneously
- NEVER start a new task until the current one has a session-end comment

### What NOT To Read At Session Start
- SOUL.md is loaded automatically — don't re-read it
- AGENTS.md is loaded automatically — don't re-read it  
- MEMORY.md is for main sessions only — don't load in cron/heartbeat sessions
- Don't read other agents' files — stay in your scope

### What Goes Where (Document Map)
- **SOUL.md** → Your personality. Nothing else. ~50 lines max.
- **AGENTS.md** → Your permanent rules, scope, department structure. The constitution.
- **TOOLS.md** → Environment data. IPs, paths, endpoints. Reference sheet.
- **HEARTBEAT.md** → Heartbeat checks only. Not loaded in task sessions.
- **MC Task notes** → Task-specific context. Session comments. Pickup instructions.
- **memory/YYYY-MM-DD.md** → Daily log. Written at session end for major events.
