# D1.22 — Agent Communication Policy

## Primary Method: sessions_send
```
sessions_send(sessionKey="agent:TARGET:main", message="your message")
```

- Jinbe: agent:jinbe:main
- Franky: agent:franky:main
- Sanji: agent:sanji:main
- Chopper: agent:chopper:main
- Usopp: agent:usopp:main

- Brook: agent:brook:main
- Zoro: agent:zoro:main

### When It Works
- From clean Telegram sessions (Captain tagging or agent's own Telegram session)
- From fresh sessions with no stale context

### Known Issue
If target agent's main session has stale context (accumulated heartbeat/queue-check history), the message may be ignored. Fix: heartbeat session isolation (`session: "heartbeat"` in config) prevents buildup in main session.

## Captain-to-Agent
Tag @bot in Telegram topic — always works, creates clean topic-specific session.

## Async (Task Board)
Create MC task assigned to target agent. Agent picks up on next session. Reliable, not real-time.

## Cron Delivery Fix — accountId Required (2026-03-26)

**ROOT CAUSE:** The `default` Telegram account is `enabled: false`. Crons without 
`accountId` fall back to disabled default → "Outbound not configured for channel: telegram."

**RULE: Every cron with Telegram delivery MUST include `accountId` matching the agent's bot.**

```
delivery: {
  mode: "announce",
  channel: "telegram",
  to: "-1003814818004:topic:XXXX",
  accountId: "AGENT_NAME"    ← REQUIRED
}
```

Account IDs match the agent names in config:
- nami, franky, sanji, chopper, usopp, robin, brook, jinbe, zoro

**Without accountId:** delivery fails with "Outbound not configured"
**With accountId:** delivery routes through the correct bot token

This applies to ALL crons, ALL agents, ALL Telegram deliveries.
