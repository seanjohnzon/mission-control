_Last updated: 2026-03-13 | Status: Blocked (Desktop offline) | Authority: D1.1-ROADMAP.md §2_

---

## 1. Purpose

This document inventories all prior systems, workflows, prompts, automations, and infrastructure that existed before the current organization was established. Each asset is classified as **Keep**, **Adapt**, or **Retire** per D1.1 Principle 9: "Migrate before rebuilding."

---

## 2. Classification Rules

| Classification | Meaning | Action |
|---|---|---|
| **Keep** | Functional and compatible with new architecture | Migrate as-is into the appropriate execution layer |
| **Adapt** | Valuable intent, needs rewriting for new architecture | Rewrite per new standards, preserve core logic |
| **Retire** | Superseded, redundant, or broken beyond use | Archive for reference, do not migrate |

**Rule:** No asset is retired without inspection. "Probably broken" is not sufficient justification for retirement.
