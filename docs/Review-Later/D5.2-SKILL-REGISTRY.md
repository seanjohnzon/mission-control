# D5.2 — SKILL REGISTRY
**Autonomous AI Organization — OpenClaw Skill Invocation Metadata**
_Last updated: 2026-03-13 | Status: Active | Authority: D1.12-CAPABILITY_ACQUISITION_POLICY.md_

---

## 1. Purpose

This document tracks all OpenClaw skills installed on organization machines. It records invocation metadata — how to use each skill, what it does, and where it lives. Approval and compliance metadata live in D5.7-CAPABILITY-REGISTER.md.

---

## 2. Governance

**Authoritative for:** Skill invocation metadata (triggers, location, capabilities, dependencies)
**NOT authoritative for:** Approval status, risk class, cost ceilings — those live in D5.7
**Write authority:** Bridge (Conductor) may add/update skill entries after installation. Human approves new skill installations.

## 7. Skill Installation Process

1. Identify skill need (from intake lane, research, or operational gap)
2. Locate skill (clawhub.com, community, or author via skill-creator)
3. Review SKILL.md for capabilities, dependencies, and risks
4. File adoption proposal if risk > Low (per D1.12 §7.2)
5. Human approves installation
6. Install via OpenClaw CLI
7. Register in this file (invocation metadata) and D5.7 (compliance metadata)
8. Test in controlled context before production use
