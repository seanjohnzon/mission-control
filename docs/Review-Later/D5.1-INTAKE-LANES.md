# D5.1 — INTAKE LANES
**Autonomous AI Organization — How New Capabilities, Knowledge, and Methods Enter the System**
_Last updated: 2026-03-13 | Status: Active | Authority: D1.1-ROADMAP.md §9_

---

## 1. Purpose

This document defines how external information, experiments, and capability candidates enter the organization. Intake is deliberate, not reactive. Every external input must be classified, routed, and filed — not simply consumed.

---

## 2. Intake Principles

1. **Every intake has a defined lane.** Ad hoc consumption that skips routing is noise, not signal.
2. **Intake produces artifacts, not awareness.** Reading something doesn't count. Filing a brief, updating a register, or producing a recommendation does.
3. **Signal beats volume.** One well-routed research brief outweighs ten bookmarked links.
4. **Intake matures in stages.** Start manual, add structure, then automate. Do not automate before the manual pattern is stable.
5. **Cost awareness.** External research tools (Perplexity, API calls) have costs. Log usage and respect ceilings.

---

## 3. Defined Lanes

### 3.1 Research Lane

**Source:** X/Grok posts, articles, YouTube, arXiv, blogs, web search results
**Owner:** forge (Researcher)
**Output:** Research brief filed to brain (Archivist), summary posted to Telegram ops-channel
**Cadence:** On trigger or scheduled sweep
**Current maturity:** Stage 1 (Manual Intake)

**Stage progression:**
| Stage | Description | Entry Criteria |
|---|---|---|
| 1 — Manual | Human reads source, pastes summary/link to Telegram. Researcher processes on demand. | Default |
| 2 — Structured | Human uses intake template. Researcher files, tags, routes automatically. Brief → Archivist. | Stage 1 stable, template defined |
| 3 — Automated | Monitored feeds (RSS, X lists, YouTube channels) swept on schedule. Researcher filters and files. Human reviews digest. | Stage 2 stable, feed list approved |

### 3.2 Experiment Lane

**Source:** Human-initiated experiments with new tools, models, or methods
**Owner:** Human + forge
**Output:** Result doc filed in D5.4-EXPERIMENT-REGISTRY.md, pass/fail verdict, adoption decision
**Cadence:** As initiated
**Current maturity:** Active (experiments logged as they happen)

### 3.3 Skill/Plugin Lane

**Source:** OpenClaw skills marketplace (clawhub.com), community contributions, custom builds
**Owner:** Human approves, bridge installs
**Output:** New capability registered in D5.2-SKILL-REGISTRY.md and D5.7-CAPABILITY-REGISTER.md
**Cadence:** On discovery or need
**Current maturity:** Active (three built-in skills installed)

### 3.4 Model Lane

**Source:** New local or API models — new releases, community benchmarks, provider announcements
**Owner:** Human evaluates, forge benchmarks
**Output:** Model added to inference roster or rejected with reason. Recorded in D1.6 and D5.4.
**Cadence:** On discovery or scheduled review
**Current maturity:** Active (Ollama models installed via CHG-007)

### 3.5 Environment/Change Lane

**Source:** macOS updates, OpenClaw updates, hardware changes, network topology changes
**Owner:** Bridge monitors, human approves changes
**Output:** Updated runbooks, config committed to docs, change record in CHANGE-REGISTER
**Cadence:** Continuous monitoring, change-triggered
**Current maturity:** Active (CHG process operational)

### 3.6 Cost/Usage Lane

**Source:** Model usage logs, token burn rates, provider billing, recurring automation load
**Owner:** brain (Merchant / finance-facing logic) with bridge reporting usage data
**Output:** Visibility into system costs, usage concentration, optimization recommendations
**Cadence:** Reviewed per phase and when new models or automations are introduced
**Current maturity:** Not yet active (Phase 2+, requires brain agent)

## 4. Intake Routing Rules

| Input Type | Route To | Filed In | Requires Human Approval? |
|---|---|---|---|
| Research link/article | Researcher → Archivist | Research brief + D5.4 if experiment | No (filing). Yes (adoption). |
| New model release | Model Lane | D1.6 + D5.4 | Yes (installation) |
| New skill/plugin | Skill Lane | D5.2 + D5.7 | Yes (installation) |
| Tool/capability candidate | Experiment Lane | D5.4 + D5.7 (candidate) | Yes (activation) |
| macOS/OpenClaw update | Environment Lane | CHANGE-REGISTER | Yes (if risk ≥ Medium) |
| Cost anomaly | Cost Lane | Ops digest | Alert to human |
| Unsolicited external input | Drop or classify first | — | — |

---

## 5. Intake Template (Stage 2+)

When Stage 2 is reached, human intake uses this format:

```
Source: [URL or description]
Type: research | model | skill | tool | environment | cost
Priority: urgent | normal | low
Summary: [1-2 sentence description]
Action requested: file | evaluate | adopt | monitor
```

---

## 6. Cost Tracking for External Research

When Perplexity or other paid external research tools are activated (CAP-006 decision pending):
- Log each usage with timestamp, query summary, and estimated cost
- Track cumulative spend against approved ceiling
- Alert human when spend exceeds 80% of ceiling
- Do not exceed ceiling without explicit approval

**Current Perplexity status:** CAP-006 candidate. No API key provisioned. No cost ceiling set.
