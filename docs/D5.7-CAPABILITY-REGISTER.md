# D5.7 — CAPABILITY REGISTER
**Autonomous AI Organization — Master Register of All Approved Capabilities**
_Last updated: 2026-03-13 | Status: Active | Authority: D1.12-CAPABILITY_ACQUISITION_POLICY.md_

---

## Register Governance

**Authoritative for:** Approval status and compliance fields for all capability categories (CAP-MOD, CAP-SKL, CAP-AUT, CAP-SVC, CAP-PLT, CAP-WFM, CAP-INF)

**Relationship to D5.2:** D5.2-SKILL-REGISTRY.md is a specialized index for CAP-SKL invocation metadata only. D5.7 is the master compliance register. When a skill is registered, it has a full record here and a reference entry in D5.2 for invocation metadata. D5.7 is authoritative for approval and compliance. D5.2 is authoritative for invocation metadata.

**Write authority (D1.12 §8.1):**
- Bridge (Conductor) may update: `status`, `last_reviewed`, `review_due`, `notes`, cost usage tracking
- Bridge may NOT change: `approved_by`, `approval_date`, `risk_class`, `authorized_agents`, `authorized_task_classes` — these require a new human approval decision

**Schema version:** 1

---

## Field Reference

| Field | Description |
|---|---|
| `id` | CAP-### sequential (CAP-001, CAP-002, …) |
| `name` | Short human-readable name |
| `category` | CAP-MOD / CAP-SKL / CAP-AUT / CAP-SVC / CAP-PLT / CAP-WFM / CAP-INF |
| `status` | `candidate` / `approved` / `active` / `deprecated` / `retired` / `declined` |
| `risk_class` | 1 (Low) / 2 (Medium) / 3 (High) / 4 (Critical) |
| `version` | Integer starting at 1; increments on scope/risk/cost change |
| `approved_by` | `human` |
| `approval_date` | YYYY-MM-DD |
| `authorized_agents` | Which agents may use this capability |
| `authorized_task_classes` | Which task classes it may be used in |
| `cost_ceiling` | Approved monthly cost limit; null if free |
| `experiment_ref` | D5.4 experiment ID, or null |
| `source_ref` | D1.10 research packet ID, or null |
| `last_reviewed` | YYYY-MM-DD |
| `review_due` | YYYY-MM-DD (cadence: Risk 1=annual, Risk 2=6-month, Risk 3=quarterly, Risk 4=monthly) |
| `notes` | Conditions, restrictions, known issues, change_notes for version history |

## Register Maintenance

- **Bridge (Conductor)** surfaces `review_due` dates in the weekly digest
- **Human** approves all new entries, scope changes, and risk class changes
- **Review cadence:** Risk Class 1 = annual; Risk Class 2 = 6-month; Risk Class 3 = quarterly; Risk Class 4 = monthly
- **Legacy capabilities** (capabilities already in use without a register entry) are surfaced through legacy asset review (D1.12 §5.5) and must be registered or retired within 90 days of discovery
