# D1.22 — Agent Communication Policy

## Official Method: One-shot Crons with Isolated Sessions
DO NOT use sessions_send for inter-agent comms. It hits stale main sessions.

```
cron(action=add, job={
  agentId: "target-agent",
  sessionTarget: "isolated",
  deleteAfterRun: true,
  schedule: { kind: "at", at: "<1 min from now>" },
  payload: { kind: "agentTurn", message: "your message", timeoutSeconds: 120 },
  delivery: { mode: "announce", channel: "telegram", to: "-1003814818004:topic:1452" }
})
```

## Why
- Creates fresh isolated session (no stale context)
- Agent reads ONLY the message + AGENTS.md (clean)
- Delivers to Telegram so Captain can see exchanges

## Captain-to-Agent
- Tag @bot in Telegram topic → works perfectly
- Creates topic-specific session, always clean

## Task Board (async)
- Create MC task assigned to target agent
- Agent picks up on next session
- Reliable but not real-time
