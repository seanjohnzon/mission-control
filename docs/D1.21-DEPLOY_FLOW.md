# D1.21 — Deploy Flow

LOCAL = Dev/Test. REMOTE (GitHub Pages) = UAT.

## Pipeline
1. Franky → builds on feature branch (LOCAL)
2. Chopper → screenshots of CHANGED areas only (LOCAL QA)
3. PASS → Sanji → git tag (rollback point) → merge to main → deploy to REMOTE
4. Chopper → post-deploy verification on UAT
   - Screenshots of changed areas
   - Quick visual scan of each tab
   - CI/CD regression covers everything else
5. PASS → Done. FAIL → Sanji reverts to tagged version.

## Rules
- Screenshots ONLY where the change is — don't pile up unnecessary proof
- Git tag before EVERY deploy = guaranteed rollback point
- Sanji owns main branch and the revert command. Nobody else merges.
- Franky pushes to feature branches. Sanji merges to main.
