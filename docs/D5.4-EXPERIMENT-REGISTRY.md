# D5.4 — EXPERIMENT REGISTRY
**Autonomous AI Organization — Log of All Experiments, Outcomes, and Adoption Decisions**
_Last updated: 2026-03-13 | Status: Active | Authority: D1.1-ROADMAP.md §9_

---

## 1. Purpose

Every experiment the organization runs — new models, tools, methods, prompts, or workflows — is logged here with its outcome and adoption decision. This prevents re-running failed experiments, provides evidence for adoption decisions, and builds institutional knowledge.

---

## 2. Experiment Lifecycle

```
Proposed → Approved → Running → Completed → Decision (adopt / defer / reject)
```

**Rules:**
- Experiments require human approval before starting (unless pre-approved by task assignment)
- Every experiment has a defined scope, success criteria, and cost bound
- Results are filed here regardless of outcome — negative results are valuable
- Adoption decisions are recorded even when the decision is "not now"

---

## 3. Field Reference

| Field | Description |
|---|---|
| `exp_id` | EXP-### sequential |
| `title` | Short descriptive title |
| `status` | `proposed` / `approved` / `running` / `completed` / `cancelled` |
| `proposed_by` | human / bridge / forge |
| `proposed_date` | YYYY-MM-DD |
| `started` | YYYY-MM-DD or null |
| `completed` | YYYY-MM-DD or null |
| `scope` | What is being tested and how |
| `success_criteria` | How to determine if the experiment succeeded |
| `cost_bound` | Maximum cost allowed for the experiment |
| `outcome` | What happened — data, observations, measurements |
| `decision` | `adopt` / `defer` / `reject` + rationale |
| `task_ref` | TASK-### if governed by a task |
| `capability_ref` | CAP-### if the experiment evaluates a capability |
| `notes` | Additional context, lessons learned |
